## Slide 7: Technical Excellence & Efficiency
### 📌 Slide Title:
**Engineered for Speed, Reliability, and Safety**

### 📌 Layout:
Three key architectural highlights with speed metrics

### 📌 Slide Content (Copy-Paste):
* **Single-Call API Pipeline:** 
  * Current check-in, 3-day history, and profile context are grouped into one Gemini prompt, returning structured JSON.
* **Asynchronous Backgrounding (`after()`):**
  * Generation of weekly pattern summaries runs in the background *after* response delivery, reducing frontend load times.
* **Direct Server Imports:** 
  * Removed internal API loopbacks, avoiding network roundtrips and hosting dependencies.
* **99%+ Code Quality & Efficiency:**
  * Clean, strictly-typed Next.js 15 project with zero linter errors. Optimized queries save database roundtrips on chat endpoints.

### 🖼️ Visual Ideas:
* Simple flow chart showing: Client Request ➡️ Route Handler ➡️ Asynchronous processing (`after()`) in background | Fast response to client.
* Speed badges: `Response Time < 5s (Analysis)`, `Chat Streaming < 1s`.

### 🗣️ Speaker Notes:
> *"We optimized the code to run efficiently. Instead of making multiple calls to the AI, we bundle the journal, metrics, and history into a single, high-speed call that returns structured JSON. Using modern Next.js features like 'after()', we process weekly summaries asynchronously in the background. The user gets their check-in results in under 5 seconds, and chat streaming starts in under 1 second."*

## Slide 8: Safety & Privacy First
### 📌 Slide Title:
**Safety Intercept: Putting Students First**

### 📌 Layout:
Two columns (Dual-Layer Safety Net vs. Data Privacy)

### 📌 Column 1: Dual-Layer Safety (Copy-Paste):
* **Layer 1: Client-Side Regex (Instant):**
  * Scans journal entries instantly for crisis keywords before any API request is sent. Works offline.
* **Layer 2: AI Safety Analysis:**
  * Gemini evaluates entries for subtle, indirect cries for help (e.g., *"I just want everything to stop"*).
* **Crisis Overlays:**
  * If triggered, standard dashboards lock, displaying verified Indian helpline numbers (iCall, Vandrevala Foundation, AASRA).

### 📌 Column 2: Data Security (Copy-Paste):
* **Row-Level Security (RLS):**
  * Supabase DB policies enforce absolute user isolation. A student's journals are only visible to them.
* **Server-Side Security:**
  * All Gemini API keys are secured in backend environments—never exposed to the client.

### 🖼️ Visual Ideas:
* A graphic depicting a shield representing the safety layers.
* Logo icons of Supabase and Gemini AI.

### 🗣️ Speaker Notes:
> *"When dealing with mental health, safety is not a footnote—it is the core requirement. We built a dual-layer safety intercept. If a student inputs crisis keywords, or if the AI detects indirect expressions of self-harm, the app immediately suspends normal metrics and opens a calming screen with direct, clickable links to verified Indian helplines. Additionally, we enforce Supabase Row-Level Security, guaranteeing absolute privacy for their journals."*
